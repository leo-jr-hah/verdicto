## Product

- User submits an asset â†’ AI agents independently value it â†’ jurors evaluate credibility â†’ verdict stored on-chain as a composable primitive
- **Core narrative**: "The GPS for Asset Prices" - any Casper dApp can query the Oracle via cross-contract call

## What Verdicto Does (the actual product flow)
1. **User submits asset** - real estate, fine art, or commodity + their asking price
2. **x402 payment** - 2.5 CSPR assessment fee collected via micropayment
3. **Dual independent valuation** - two AI agents value the same asset using different methodologies:
   - **Comps Specialist (Agent A)** - comparable sales, market transactions, price-per-sqft
   - **DCF Specialist (Agent B)** - discounted cash flow, intrinsic value, fundamentals
4. **Juror deliberation** - three specialized jurors evaluate which valuation is more credible:
   - **Evidence Analyst** - data quality, proof strength, source reliability
   - **Market Data Interpreter** - market trends, macro context, pricing signals
   - **Precedent Researcher** - historical assessments, comparable outcomes
5. **On-chain certificate** - final assessment + all reasoning committed to Casper blockchain via HMAC receipt chain
6. **Result**: a verifiable, tamper-proof assessment certificate showing the asset's AI-determined fair value

## Verdicto Points (Competition Feedback - Implemented)
1. **LTV â†” Trust Scores** - `calculateLTV()` uses continuous formula: `ltv = base + (max - base) Ă— confidence Ă— valueRatio`. Low confidence â†’ visibly lower LTV. Trust breakdown exposed in API + frontend.
2. **Escrow On-Chain** - Escrow lock (platform â†’ borrower) and release (borrower â†’ platform) produce real on-chain tx hashes. Visible in loan card. **Note: collateral asset is tracked off-chain (database record). Only the CSPR loan amount is on-chain. This is explicitly scoped as MVP - no on-chain collateral token lock.**
3. **Revaluation = Juror Deliberation** - Revalue re-runs dual-agent deliberation, produces HMAC receipt chain, stores commitment on-chain. Full reasoning exposed. Margin call = deliberated AI judgment.
4. **x402 on Money Flow** - Protocol-correct HTTP 402 on all money-flow endpoints: `/api/loans/create` (5 CSPR), `/api/loans/:id/repay` (2.5 CSPR), `/api/insurance/create` (3 CSPR), `/api/insurance/:id/claim` (3 CSPR). Returns `x402Version: "2"`, `paymentRequirements` with `scheme`, `supportedChains`, `payTo`, `maxAmountRequired`. Verified live with curl.
5. **Sequencing** - Escrow lock before revaluation logic.
6. **Auto-Revaluation Monitor** - `setInterval` checks active loans every 5 min. If >30 min since last revaluation, triggers dual-agent deliberation pipeline automatically. System monitors its own loans.
7. **Demo Mode** - `DEMO_MODE=true` env flag: checks every 30s with 0 staleness threshold (fires on every check). For live demos showing autonomous behavior.
8. **Admin Endpoints** - `POST /api/admin/force-revalue/:loanId` (manual revaluation trigger), `GET /api/admin/loans` (list all loans with revaluation metadata). Gated by `x-admin-secret` header matching `ADMIN_SECRET` env var (403 if missing/wrong). For demo/testing only.
