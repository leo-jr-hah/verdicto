# Verdict — Project Context

## Product
Verdict is an **autonomous RWA asset assessment and valuation platform**.
- User submits an asset → AI agents independently value it → jurors evaluate credibility → user gets an on-chain assessment certificate
- The product is: **"What is this asset actually worth, verified by AI and recorded on-chain?"**
- Assessment fee: 2.5 CSPR via x402 micropayment

## Architecture
- **dashboard/** — React + TypeScript + Vite frontend (React 19, Vite)
- **agents/** — Backend: orchestrator + 2 valuation agents + 3 jurors + trust framework + x402 micropayments
- **contracts/** — Rust smart contracts (VotingContract, ReputationRegistry) via Odra framework
- Deployed on: GCP via Cloud Run + Firebase hosting

## What Verdict Does (the actual product flow)
1. **User submits asset** — real estate, fine art, or commodity + their asking price
2. **x402 payment** — 2.5 CSPR assessment fee collected via micropayment
3. **Dual independent valuation** — two AI agents value the same asset using different methodologies:
   - **Comps Specialist (Agent A)** — comparable sales, market transactions, price-per-sqft
   - **DCF Specialist (Agent B)** — discounted cash flow, intrinsic value, fundamentals
4. **Juror deliberation** — three specialized jurors evaluate which valuation is more credible:
   - **Evidence Analyst** — data quality, proof strength, source reliability
   - **Market Data Interpreter** — market trends, macro context, pricing signals
   - **Precedent Researcher** — historical assessments, comparable outcomes
5. **On-chain certificate** — final assessment + all reasoning committed to Casper blockchain via HMAC receipt chain
6. **Result**: a verifiable, tamper-proof assessment certificate showing the asset's AI-determined fair value

## Backend Agent Architecture (verified against code)
- **Orchestrator** (`agents/orchestrator/index.ts`) — receives assessment requests, fetches data, dispatches agents
- **Valuation Agent A (Comps Specialist)** — comparable sales via RentCast API
- **Valuation Agent B (DCF Specialist)** — DCF analysis via FRED economic data
- **Evidence Analyst** — data validation via MiMo LLM
- **Market Data Interpreter** — market context via MiMo LLM + live feeds
- **Precedent Researcher** — precedent search via Vectra RAG + vector DB
- **Trust Framework** (`agents/shared/trust-framework.ts`) — executionScore (40%) + outputConsistency (30%) + economicStake (30%) → aggregate 0-1000
- **Tiers**: unverified / bronze / silver / gold / platinum
- **Verdict choices**: `agent_a_preferred | split_fifty | agent_b_preferred`
- **HMAC receipt chain** (`agents/shared/audit-trail.ts`) — SHA-256 + HMAC-SHA256
- **x402 micropayments** (`agents/shared/x402-middleware.ts`) — real on-chain verification via CSPR.cloud
- **Assessment fee**: 2.5 CSPR per assessment
- **Lending fee**: 5 CSPR per loan (x402 gated)
- **Insurance fee**: 3 CSPR per policy (x402 gated)

## Verdict Points (Competition Feedback — Implemented)
1. **LTV ↔ Trust Scores** — `calculateLTV()` uses continuous formula: `ltv = base + (max - base) × confidence × valueRatio`. Low confidence → visibly lower LTV. Trust breakdown exposed in API + frontend.
2. **Escrow On-Chain** — Escrow lock (platform → borrower) and release (borrower → platform) produce real on-chain tx hashes. Visible in loan card.
3. **Revaluation = Juror Deliberation** — Revalue re-runs dual-agent deliberation, produces HMAC receipt chain, stores commitment on-chain. Full reasoning exposed. Margin call = deliberated AI judgment.
4. **x402 on Money Flow** — x402 on repay endpoint. Disbursement uses real CSPR transfers.
5. **Sequencing** — Escrow lock before revaluation logic.

## Wallet Integration (CRITICAL)
- **Official Casper Wallet SDK** (injected by extension) — NO external npm packages needed
- `window.CasperWalletProvider` is the factory function
- `window.CasperWalletEventTypes` has event names: Connected, Disconnected, ActiveKeyChanged
- Flow: `const provider = CasperWalletProvider()` → `provider.requestConnection()` → listen for events → `provider.getActivePublicKey()`
- **CRITICAL: provider.on() does NOT exist!** Events are DOM CustomEvents on `window`
- **Payment flow:** `signPayment()` in CSPRClickContext creates a native transfer deploy via `casper-js-sdk`
- Chrome Web Store: `https://chromewebstore.google.com/detail/casper-wallet/abkahkcbhngaebpcgfmhkoioedceoigp?hl=en`
- Testnet Faucet: `https://testnet.cspr.live/tools/faucet`
