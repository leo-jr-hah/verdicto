## SDK Integration (Working — 2026-06-15)

### casper-js-sdk v5.0.12 — Correct API
```ts
import { CasperNetwork, RpcClient, HttpHandler, PrivateKey, KeyAlgorithm } from 'casper-js-sdk';

const rpcClient = new RpcClient(new HttpHandler('https://node.testnet.casper.network/rpc'));
const network = await CasperNetwork.create(rpcClient);

// Transfer
const tx = network.createTransferTransaction(
  senderPublicKey, recipientPublicKey,
  'casper-test', '10000000000', 100_000_000, 1800000, Date.now()
);
tx.sign(privateKey);
const result = await network.putTransaction(tx);

// Contract call
const tx = network.createContractCallTransaction(
  senderPublicKey, contractHash, entryPoint,
  'casper-test', 100_000_000, 1800000, args
);
```

### Key Learnings
- **DO NOT use** `makeCsprTransferTransaction()` or `makeCsprTransferDeploy()` — broken in v5.x
- **USE** `network.createTransferTransaction()` and `network.createContractCallTransaction()`
- **Hash extraction:** `result.transactionHash` is a custom object — use `JSON.parse(JSON.stringify(result.transactionHash))` to get the hex string
- **Min transfer:** 10 CSPR (10_000_000_000 motes) — 1 CSPR triggers "insufficient transfer amount"
- **Args API:** `Args.insert()` crashes with `Cannot read properties of undefined (reading 'set')` — use `network.createTransferTransaction()` which handles args internally
- **Balance query:** `network.queryLatestBalance()` has a serialization bug — use raw fetch to `query_balance` RPC with `{purse_identifier: {main_purse_under_public_key: hex}}`
